## Описание программы

*Training Planner — это GUI‑приложение для планирования и учёта тренировок. Программа помогает пользователям систематизировать информацию о спортивных занятиях, анализировать прогресс и эффективно планировать будущие тренировки.*


Основные функции:
* добавление тренировок с указанием даты, типа и длительности;
* просмотр всех тренировок в виде удобной таблицы;
* фильтрация записей по типу тренировки и дате;
* автоматическое сохранение данных между запусками в формате JSON;
* валидация вводимых данных для предотвращения ошибок.

## Примеры использования

### Сценарий 1. Добавление первой тренировки


**Действие:**
1. Запустить приложение `python app.py`.
2. В поле **«Дата (ГГГГ‑ММ‑ДД)»** ввести: `2026-04-25`.
3. В поле **«Тип тренировки»** ввести: «Бег».
4. В поле **«Длительность (мин)»** ввести: `45`.
5. Нажать кнопку **«Добавить тренировку»**.

**Результат:**
* появляется всплывающее сообщение: `✅ Тренировка добавлена!`;
* в таблице появляется новая запись:


| Дата | Тип | Длительность (мин) |
|------|---------|------------------|
| 2026-04-25 | Бег | 45 |

---

### Сценарий 2. Добавление нескольких тренировок разного типа


**Действия:**
1. Добавить тренировку: дата `2026-04-24`, тип «Плавание», длительность `60`.
2. Добавить тренировку: дата `2026-04-23`, тип «Силовая», длительность `90`.
3. Добавить ещё одну тренировку: дата `2026-04-22`, тип «Йога», длительность `30`.

**Результат:** таблица содержит 4 записи:


| Дата | Тип | Длительность (мин) |
|------|---------|------------------|
| 2026-04-25 | Бег | 45 |
| 2026-04-24 | Плавание | 60 |
| 2026-04-23 | Силовая | 90 |
| 2026-04-22 | Йога | 30 |


---

### Сценарий 3. Фильтрация по типу тренировки

**Действие:**
1. В выпадающем списке **«Тип»** выбрать «Бег».

**Результат:** в таблице отображаются только тренировки типа «Бег»:

| Дата | Тип | Длительность (мин) |
|------|---------|------------------|
| 2026-04-25 | Бег | 45 |

**Примечание:** фильтрация срабатывает мгновенно после выбора значения.

---

### Сценарий 4. Фильтрация по дате


**Действие:**
1. В поле **«Дата»** ввести `2026-04-24`.

**Результат:** в таблице отображаются тренировки за указанную дату:

| Дата | Тип | Длительность (мин) |
|------|---------|------------------|
| 2026-04-24 | Плавание | 60 |


**Примечание:** поиск по дате работает по частичному совпадению (можно ввести только год или месяц).

---

### Сценарий 5. Комбинирование фильтров

**Действие:**
1. Выбрать в фильтре **«Тип»** значение «Силовая».
2. Ввести в поле **«Дата»** `2026-04`.

**Результат:** отображаются все силовые тренировки за апрель 2026 года:

| Дата | Тип | Длительность (мин) |
|------|---------|------------------|
| 2026-04-23 | Силовая | 90 |


---

### Сценарий 6. Сброс фильтров


**Действие:** нажать кнопку **«Сбросить фильтры»**.

**Результат:** в таблице снова отображаются все имеющиеся тренировки (как в сценарии 2).

---

### Сценарий 7. Проверка валидации данных

**Тестовые случаи и результаты:**

1. **Неверный формат даты:**
   * ввод: `25-04-2026` в поле даты;
   * результат: сообщение `❌ Ошибка: Неверный формат даты! Используйте ГГГГ-ММ-ДД`.

2. **Пустое поле типа:**
   * оставить поле «Тип тренировки» пустым;
   * результат: сообщение `❌ Ошибка: Введите тип тренировки!`.

3. **Отрицательная длительность:**
   * ввод: `-30` в поле длительности;
   * результат: сообщение `❌ Ошибка: Длительность должна быть положительным числом!`.

4. **Нечисловая длительность:**
   * ввод: `сорок пять` в поле длительности;
   * результат: сообщение `❌ Ошибка: Длительность должна быть числом!`.

5. **Корректный ввод минимальной длительности:**
   * ввод: `1` в поле длительности;
   * результат: `✅ Тренировка добавлена!` (минимально допустимое значение).

---

### Сценарий 8. Сохранение и загрузка данных

**Действие:**
1. Закрыть приложение после добавления нескольких тренировок.
2. Снова запустить приложение командой `python app.py`.

**Результат:** все ранее добавленные тренировки автоматически загружаются из файла `trainings.json` и отображаются в таблице.
